# Mk2 Crossover Config — 150 Hz LR4 + 1250 Hz LR4

Target configuration for MiniDSP 4×10 HD. Enter these values into the plugin
after loading a default 2-way advanced config and adding channels.

**Status:** Pre-measurement starting point. All filters are textbook LR4 with
biquad coefficients at 96 kHz sample rate. Trim and delay values are estimates
— verify and adjust after REW measurement.

> **Quick import:** Use `generate_minidsp_xml.py` to generate a plug-and-play
> MiniDSP XML config (File → Import in the plugin). Pre-built files:
> - `mk2-150-1250-lr4.xml` — Mk2 with H2606, 150/1250 Hz
> - `mk3-sb26stac-1100hz.xml` — Mk3 fallback with SB26STAC, 150/1100 Hz
>
> Example: `python generate_minidsp_xml.py --tweeter-hp 1100 --tweeter-trim -1.0 --mid-lp 1100 > mk3-sb26stac-1100hz.xml`

#### 2. Gain trim

Set **channel gain to −5.5 dB** in MiniDSP mixer/volume tab. This compensates
for the H2606's ~95.2 dB sensitivity in the WG212 vs the 15W/4434G00's
89.7 dB sensitivity. Start at −5.5 dB and adjust after REW sweep. The
real_datasheet_response simulation shows the combined system at ~−8.8 dB overall
in the treble region (includes filter effects), so expect some fine-tuning.

## Delay

Estimated acoustic center offsets from baffle plane:

| Driver | Depth behind baffle | Delay needed |
|--------|--------------------|--------------|
| H2606 in WG212 | ~15 mm | **0 ms (reference)** |
| 15W/4434G00 | ~55 mm | Delay by −0.12 ms (set delay on mid, not tweeter) |
| GRS 8SW-4HE-8 | ~80 mm | Delay by −0.19 ms |

**Alternative approach:** delay the tweeter forward (add 0.12 ms to tweeter).
MiniDSP only adds delay, it can't add negative delay. So:

| Channel | Delay entry |
|---------|-------------|
| Woofer | 0.19 ms |
| Midrange | 0.12 ms |
| Tweeter | 0 ms (reference) |

This aligns all acoustic centers. **Verify with REW step response** — adjust
until impulse response shows a single clean peak.

LR4 acoustic slopes sum to in-phase at crossover. If the measured response shows

should be checked for absolute polarity (+ on binding post to + on driver).

## Verification Workflow

1. Load config into MiniDSP plugin → upload to device
2. REW sweep with UMIK-1 at 1 m, on-axis tweeter
3. Check:
   - Crossover null or notch? → adjust polarity or delay
   - Tweeter too hot/cold? → adjust gain trim
   - Response not flat? → add PEQ after crossover filters
4. Repeat at 15° and 30° horizontal for directivity match
5. Nearfield woofer + mid for port/leakage check

## Notes

- These coefficients assume **96 kHz** MiniDSP sample rate (default for 4×10 HD).
  If the plugin is set to 48 kHz, don't use these — regenerate them.
- The Linkwitz Transform is per-speaker (one per woofer pair, applied to both
  Out 1 and Out 2 identically). If the woofers are wired in series (single
  amplifier channel), apply the LT once.
- If using a single amp channel for both woofers (series wiring → 8 Ω), change
  to 3-channel mode: Out 1 = woofer, Out 2 = mid, Out 3 = tweeter.
- If using separate amp channels per woofer, the 2 woofer outputs can have
  slightly different delays to compensate for physical offset in push-push
  placement (verify with measurement).
